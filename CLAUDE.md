## Что строим

**Mentoria Hub** — рабочий MVP EdTech-платформы для хакатона («Working MVP Challenge»).
Платформа, где школьники находят образовательные возможности (олимпиады, стажировки, стипендии, конкурсы) и проходят курсы Mentoria **асинхронно**, без живых занятий.

**Главный принцип:** это НЕ лендинг. Должен ощущаться как первая версия настоящего продукта с реальным пользовательским путём. Mock-данные ок, но путь должен быть живым и интерактивным.

**Целевая аудитория:** ученики 8–11 классов (Казахстан и другие страны), готовящиеся к поступлению, ищущие олимпиады/хакатоны/стажировки/стипендии и доступные курсы.

## Стек (зафиксирован)

- **Backend:** Python 3.12 + **Django 6.0.6** (Django 6.0 поддерживает Python 3.12/3.13). Встроенная admin-панель закрывает требование F, встроенная авторизация — профили учеников.
- **Frontend:** Django-шаблоны + **Tailwind через CDN** (без сборки — экономит время, выглядит современно). UX важен (20% оценки).
- **Данные:** SQLite + сиды mock-данных (фикстуры или management-команда).
- **Деплой:** Render / Railway / PythonAnywhere (проще для Django, чем Vercel/Netlify).

## Обязательные функции MVP (ядро оценки — сделать ВСЕ 6)

- [ ] **A. Главная страница** — что такое Mentoria Hub, для кого, ценностное предложение (возможности + асинхронные курсы), кнопки «Найти возможности» / «Начать обучение».
- [ ] **B. Каталог возможностей** — карточки (название, категория, формат, дедлайн, описание, требования, кнопка «Подать заявку»). Фильтры по категории / классу / формату / направлению (Бизнес, STEM, Соц. влияние, Финансы, Программирование, Наука). Функция «Сохранить / в избранное». Минимум **8–10** возможностей.
- [ ] **C. Курсы** — минимум **2–3** курса, в каждом несколько уроков. Курс: название, описание, уровень, список уроков, материалы, видео-плейсхолдер, мини-тест, прогресс-бар.
- [ ] **D. Личный кабинет ученика** — сохранённые возможности, записанные курсы + прогресс, ближайшие дедлайны, рекомендации.

- [ ] **F. Админ-панель** — добавление/редактирование/удаление возможностей и курсов. Django admin закрывает это почти бесплатно.

## Эскиз моделей данных (Django)

- **StudentProfile** (OneToOne к User): grade (8–11), interests (теги), subjects, goals
- **Opportunity**: title, category, direction (Business/STEM/Finance/...), format, deadline, description, requirements, apply_url, grade_level, tags
- **SavedOpportunity**: user → opportunity (избранное)
- **Course**: title, description, difficulty_level
- **Lesson**: course (FK), title, content, video_url, order
- **Quiz / MiniTask**: привязка к lesson или course, вопросы
- **Enrollment**: user → course, progress (%)
- **LessonProgress**: user → lesson, completed (bool)

## Пользовательский путь для демо

1. Ученик открывает Mentoria Hub → регистрируется → выбирает интересы (Бизнес, STEM, английский, поступление).
2. Платформа показывает рекомендованные возможности и курсы.
3. Ученик сохраняет хакатон и летнюю программу в избранное.
4. Открывает курс «Английский для академического успеха», проходит первый урок + мини-тест.
5. Личный кабинет обновляет прогресс и показывает ближайшие дедлайны.
6. Админ через админ-панель добавляет новую олимпиаду или курс.

## ПЛАН ПО ФАЗАМ

### Фаза 1 — Скелет проекта и данные (День 1, первая половина)
1. `startproject config .` + `startapp core`, добавить `core` в INSTALLED_APPS.
2. Описать модели по эскизу выше → `makemigrations` → `migrate`.
3. Зарегистрировать все модели в `core/admin.py` (закрывает требование F).
4. `createsuperuser`.
5. Засеять mock-данные: 8–10 возможностей разных направлений + 2–3 курса с уроками и мини-тестами.
- **Контроль:** `runserver`, зайти на `/admin`, увидеть и поредактировать возможности/курсы.

### Фаза 2 — Студенческая часть (День 1 вторая половина → День 2)
6. Главная (ценностное предложение + кнопки).
7. Каталог возможностей + фильтры + «Сохранить в избранное».
8. Список курсов → страница курса → прохождение урока + мини-тест → прогресс-бар.
9. Личный кабинет (сохранённое, мои курсы + прогресс, дедлайны).
- Стиль: Tailwind через CDN.
- **Контроль:** пройти весь путь руками: открыл → каталог → сохранил → начал курс → прошёл урок → прогресс в кабинете.

### Фаза 3 — Онбординг и рекомендации (День 2)
10. Регистрация + онбординг на встроенной авторизации Django → сохранить класс/интересы/предметы/цели в StudentProfile.

- **Контроль:** новый ученик с интересом «Бизнес» → видит релевантные карточки.

### Фаза 4 — Полировка и бонусы (День 2 вечер → День 3 утро)
